**Root cause:** Zod v4 requires an explicit key schema. `z.record(z.any())` (7 occurrences) is now a 1-arg call to a 2-arg function (TS2554), and its `Record<any, unknown>` inference breaks two downstream `url.build` / `url.buildQuery` calls.

**Files:**
- Modify: `electron/mcp/tools.ts`

- [x] **Step 1: Replace every occurrence**

Replace all 7 occurrences of `z.record(z.any())` with `z.record(z.string(), z.any())`. They are at (approx) lines 547, 561, 578, 1901, 1927, 2384, 2411 — but find them by exact string, do not trust line numbers:

Find: `z.record(z.any())`
Replace: `z.record(z.string(), z.any())`

The inferred type becomes `Record<string, any>`; `any` is assignable to the stricter `UrlBuildInput`, so the two `url.*` errors (lines ~2390, ~2414) resolve automatically.

### Task 6: I1 — convo poller cleared synchronously on kill

**Problem:** `captureConversationId` starts a `setInterval` whose handle is local; it self-clears only on the next tick (≤1s) via `!this.sessions.has(id)`. Store the handle and clear it synchronously in `kill`/`killAll` so a just-killed terminal can never emit a `convo` event.

**Files:**
- Modify: `electron/services/terminals.ts`
- Test: `electron/services/terminals.test.ts`

- [ ] **Step 1: Write the failing test**

Add to `terminals.test.ts` (follow the existing import/setup style in that file):
```ts
it("clears the convo poller immediately on kill (no convo event after kill)", () => {
  vi.useFakeTimers()
  const svc = new TerminalService()
  // create a terminal (use the same construction the other tests use)
  const info = svc.create()
  const events: string[] = []
  svc.onEvent((e) => events.push(e.type))
  svc.kill(info.id)
  // advance well past the 1s poll interval; a leaked timer would fire here
  vi.advanceTimersByTime(5000)
  expect(events).not.toContain("convo")
  vi.useRealTimers()
})
```

- [ ] **Step 2: Run it — expect FAIL** (or flakiness from the leaked timer)

Run: `npx vitest run electron/services/terminals.test.ts -t "clears the convo poller"`

- [ ] **Step 3: Implement**

In `TerminalService`, add a field near the other private maps:
```ts
private convoTimers = new Map<string, ReturnType<typeof setInterval>>()
```
In `captureConversationId`, store the handle and drop it on clear:
```ts
const timer = setInterval(() => {
  attempts++
  const convoId = resolveTranscriptId(this.ccProjectsRoot, cwd, spawnedAt, preexisting)
  if (convoId) {
    this.emitEvent({ type: "convo", id, ccConversationId: convoId })
    clearInterval(timer)
    this.convoTimers.delete(id)
  } else if (attempts >= 10 || !this.terminals.has(id)) {
    clearInterval(timer)
    this.convoTimers.delete(id)
  }
}, 1000)
this.convoTimers.set(id, timer)
```
In `kill(id)`, before/after deleting from the map, clear the poller:
```ts
const t = this.convoTimers.get(id)
if (t) { clearInterval(t); this.convoTimers.delete(id) }
```
Do the same inside `killAll()` (iterate `this.convoTimers` and clear each, then `this.convoTimers.clear()`).

- [ ] **Step 4: Run the test — expect PASS**

Run: `npx vitest run electron/services/terminals.test.ts -t "clears the convo poller"`
