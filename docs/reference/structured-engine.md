# Session Spawning & the Structured (Headless) Engine

## Spawn wrappers

Sessions spawn Claude via shell wrapper for PATH resolution:
- Windows: `powershell.exe -NoLogo -NoProfile -Command claude --dangerously-skip-permissions --mcp-config {path}`
- macOS/Linux: `bash -l -c "claude --dangerously-skip-permissions --mcp-config {path}"`

The `--mcp-config` flag auto-connects Claude to the ClaudeTUI MCP server so Claude can control the app.

**Auto-restore on startup:** All persisted sessions and terminals are automatically restored in parallel on app launch. Each terminal gets a fresh PTY with `--resume <conversationId>` so Claude picks up the same conversation. A "Restoring session..." overlay shows on each terminal until its PTY connects.

## Headless engine (BO-1..BO-5) + input/permissions (BO-3) + the live engine switch (BO-4a)

`TerminalService.createHeadless` spawns `claude -p` with the stream-json transport (`HEADLESS_FLAGS` in `electron/services/streamProtocol.ts`) instead of an interactive PTY. **BO-4a wired the switch:** `create()` routes to `createHeadless` when the resolved engine is `structured` (config `rendering.engine`, set via `TerminalService.setEngine` from `ipc.ts` using `resolveRenderingEngine`). **CAPP-39 gate ④ flipped the DEFAULT to `structured`:** `resolveRenderingEngine`/`resolveEngine`/`TerminalService`'s engine field now resolve to structured **unless** config explicitly says `engine: "xterm"` (a hard cutover — solo-dev project, low risk). `xterm` is the opt-in legacy PTY path (byte-unchanged when selected); the per-terminal raw-view escape hatch (`setTerminalEngine` / the AgentView "Raw view" button) remains the per-terminal way back to xterm, and the command palette carries a rollback write-path ("Default new terminals to structured / to raw terminal (xterm)" → `config:set-rendering-engine`, persists + applies to the next-spawned terminal). The LIVE renderer fork is PER-TERMINAL — it keys on the backend-stamped `t.engine` field (`src/App.tsx` / `src/components/SplitView.tsx`) to pick `AgentView`+`AgentComposer`+`PermissionPrompt` vs `TerminalPane`, NOT a global config read (the BO-2 `agentViewFlag` dev gate is retired; `src/lib/renderingEngine.ts`'s `resolveEngine` is a kept-in-sync pure helper, not currently wired into the live fork). `list()`/`getActivity()` include headless terminals so the activity snapshot sees them. End-to-end live-verified against real `claude -p` — see `docs/spikes/bo4a-engine-switch.md`.

Two halves of human↔agent I/O on the headless path:

- **Input** — `src/components/AgentComposer.tsx` (a multiline composer, Enter=send / Shift+Enter=newline, drop-to-attach images via `saveDroppedImage`) sends its `{ text, attachments }` via `window.api.sendAgentInput` → `agent:send-input` IPC → **`TerminalService.submitAgentInput`** (CAPP-130 — the ONE entry point). It never touches `writeToSession`/pty.write. **THE SERVICE DECIDES queue-vs-send** (this kills the old renderer↔service race where the turn ended between the renderer's busy check and the IPC arriving, so the old "never send while busy" guard is retired): while the FOREGROUND is busy — `entry.foregroundActive === true` (the window from the stdin write to the turn's `result`) **OR** a parked permission (`hasPendingPermission`) — the RAW payload is ENQUEUED per terminal (`terminal:agent-queue-changed` snapshot → the composer's removable chip row); otherwise it submits immediately through the shared `submitAgentInputNow` path (slash-command classification + `agentMessageFromInput` → the `sendAgentMessage` stdin sink, OR a native `/config`-`/resume` app affordance via `ui:slash-command`). Queued messages auto-flush FIFO **one per turn**: at each `result` (foreground now idle, no permission pending — never on a permission park, since stdin buffers unread) the service dequeues ONE and re-submits it through the SAME shared path, so a queued `/compact` routes exactly like a fresh one; the flushed send re-arms `foregroundActive`, so the next `result` flushes the next. Queues are **in-memory only** (a restart drops them — auto-firing queued turns at app launch would surprise the user), dropped on kill, and **transferred across every respawn re-point** (`transferAgentQueue` — Stop/interrupt, restart, model/effort/ultracode switch, engine switch, handoff all mint a new terminal id; the queue rides onto the fresh idle proc and boots its head). Programmatic sends (scheduler, `waitForIdle` inject, `write()`) call `sendAgentMessage` directly and BYPASS the queue.
- **Permissions** — headless spawn adds `--permission-prompt-tool mcp__claudetui__approve_tool` (NOT `--dangerously-skip-permissions`). When a tool isn't pre-approved, Claude Code synchronously calls our `approve_tool` MCP tool (`electron/mcp/tools/permissions.ts`) and blocks. It attributes to the caller's terminal (identity token), `TerminalService.requestPermission` raises a tier-2 `asked` (via `markAwaitingPermission`) + pushes a `PermissionRequest` to the renderer's `PermissionPrompt` (`src/hooks/usePermissions.ts`), and blocks until the user Allow/Denies (`resolvePermission`). **Wire shapes are live-captured** in `docs/spikes/bo3-permission-prompt.md` + `permissionWire.fixtures.ts`: the tool receives `{ tool_name, input, tool_use_id }`; ALLOW MUST return `{"behavior":"allow","updatedInput":{…}}` (a bare allow is rejected and the tool is blocked); DENY is `{"behavior":"deny","message":"…"}`. "Always allow `<tool>`" persists to `<cwd>/.claude/settings.local.json` `permissions.allow` (honored by Claude Code's default setting sources next spawn); `--allowedTools` pre-approval skips the gate. If a terminal exits while a permission is pending, it's resolved as `deny("agent-exited")` so the MCP call never hangs.

## Slash commands + skills in the structured composer (BO-7 / CAPP-41)

Going headless does NOT lose skills/commands — plain `claude -p` loads user + plugin skills, custom commands, subagents, MCP, and SessionStart hooks by default, and typed `/skill`/`/command` ALREADY expand end-to-end over the stdin sink (the BO-3 input path is exactly the shape Claude expects). BO-7 adds the picker UX + native routing on top:

- **Init catalog** — the parser (`streamEvents.ts`) retains the `init` event's `slash_commands` + `skills` arrays (added to the `init` `StreamEvent` + the `AgentCatalog` type in `streamProtocol.ts`). `TerminalService` stores them per headless terminal and exposes `getCatalog(id)` → `agent:catalog` IPC → `window.api.getAgentCatalog`. (Init arrives AFTER the first user message on the stream-json path.) **CAPP-126:** the catalog is PERSISTED on the `TerminalRef` (like `resolvedModel` — both captured off the same init watch in `sessions.ts`; `recordCatalog` no-ops when unchanged) and re-seeded onto the fresh PTY on restore via `TerminalService.seedCatalog` (fills only an empty entry — a live init always wins), so a restored terminal's picker works immediately with last session's catalog. `getCatalog` returns `{ …catalog, live }` where `live` = a live init arrived THIS spawn (`sawInit`) — freshness is a property of the SPAWN, not of the picker component mount (the composer remounts on every terminal switch).
- **`/`-autocomplete picker** — `src/components/AgentComposer.tsx` mounts `useSlashPicker` (`src/hooks/useSlashPicker.ts`) + `SlashCommandPicker.tsx`; pure model in `src/lib/slashCatalog.ts`. It sources the filterable list from the LIVE init catalog (pulled on mount + tracked off live `init` stream events), NOT a hardcoded set — plus (CAPP-126) an always-available BUILTIN FLOOR (`/clear /compact /config /context /resume`, merged by `buildPickerEntries` with the live catalog winning on de-dupe) so even a never-initialized terminal offers the core commands, and a muted visible-text footer hint ("from last session — refreshes after the first reply") while running on the persisted/builtin set (keys on `getCatalog`'s `live` flag, cleared by a live init this spawn). Up/Down/Enter/Tab select, Esc dismisses; selecting inserts `/name ` (structured-only, since the composer only mounts for structured terminals).
- **Built-in routing** — the `agent:send-input` intercept (`terminal-handlers.ts`) classifies input via the pure routing table in `electron/services/slashCommands.ts` BEFORE folding for stdin: native-mapped built-ins (`/config` → theme/config cycle, `/resume` → the Ctrl+Shift+H handoff) fire an existing app affordance via the `ui:slash-command` renderer event (`App.tsx` handler reuses `cycleTheme`/`handleHandoff`) instead of being sent to Claude; everything else (`/clear`, `/compact`, `/context`, skills, plugin skills, custom commands, prose) forwards UNCHANGED — the slash is preserved — so Claude expands it. `/model` is intentionally NOT routed here (owned by BO-6 / CAPP-40 — see the `BO-6 HOOK` comments in `slashCommands.ts` + `terminal-handlers.ts` + `App.tsx`). Hermetic e2e: `fakeStream.ts`'s canned init carries a sample catalog; `e2e/structured.spec.ts` covers the picker render + the `/config` native map + the CAPP-126 pre-init builtin floor/staleness-hint case.

## Context meter (CAPP-127)

A slim, statically-visible segmented bar between AgentView and the composer (structured terminals only; hidden until the first usage-bearing `result`) showing how full the model's context window is. Pure derivation in `src/lib/contextMeter.ts` (`contextMeterFromBlocks`): a result's footprint = `input + cache_read + cache_creation` tokens (output excluded; usage-less results SKIPPED, never counted as 0); the three segments — baseline (FIRST result ≈ system prompt + tools + CLAUDE.md) / history / lastTurn — tile the total exactly and clamp ≥0 (context can SHRINK after `/compact` or `--resume`); cap = 200k, or 1M when `[1m]` appears in the alias OR the resolved id; the remainder tints warning ≥70%, danger ≥90%. Derived in `AgentView` from the FULL folded blocks (NOT the windowed tail — so a restored transcript shows the meter immediately) and lifted to `AgentSurface` ONLY when the meter's values change — a per-delta lift would setState in the parent (re-rendering the composer) on every streamed token, the same perf class CAPP-103 existed for. CLICKING the bar (not hover) opens a legend popover (`src/components/ContextMeterBar.tsx`, `position:fixed` so `.terminal-container`'s overflow can't clip it; closes on click-outside/Escape): per-color token counts, "X of Y (Z%)", an estimate caveat, and two visible TEXT actions — **Compact** (sends `/compact` through `sendAgentInput`, disabled while busy) and **Handoff** (`window.api.handoffTerminal` — the same backend mechanism as Ctrl+Shift+H, targeting ITS OWN terminal, which is correct in split-pane).
